# S35 Try 기반 제품 등록 API/UI 계획

## 목표
- 프로젝트에서 검증한 Try를 기존 제품/처방 자산으로 등록할 수 있게 한다.
- 기존 제품 기반 프로젝트 시작 흐름의 반대 방향인 `프로젝트 결과 -> 제품 자산` 흐름을 만든다.

## 범위
- `POST /projects/tries/:tryId/product` API 추가
- 서버가 Try, 그룹, 프로젝트, 원료 정보를 읽어 제품과 제품 처방을 생성
- 생성된 제품에 감사 로그 기록
- 프로젝트 상세 화면에 제품 등록 폼 추가
- API-backed Try는 promote API를 호출하고, 로컬 Try는 로컬 안내만 표시

## 제외
- 제품 승인/검토 워크플로
- 프로젝트와 제품 간 영구 관계 컬럼 추가
- 제품 등록 후 자동 라우팅
- 다중 Try 병합 제품 생성

## 구현 순서
1. API 서비스/컨트롤러 실패 테스트 추가
2. 웹 제품 등록 실패 테스트 추가
3. 실패 테스트 확인
4. API Try 기반 제품 생성 구현
5. 웹 제품 등록 폼 구현
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

## 성공 기준
- 연구자는 프로젝트 상세 화면에서 대상 Try와 제품명을 선택/입력해 제품으로 등록할 수 있다.
- API-backed Try 등록 시 `POST /projects/tries/:tryId/product`가 호출된다.
- API는 Try의 원료, 제형, 메모를 제품 처방으로 복사한다.
- 제품 생성 감사 로그에 원본 Try, 그룹, 프로젝트 정보가 남는다.
- API 실패 또는 로컬 Try는 기존 로컬 안내를 표시한다.
