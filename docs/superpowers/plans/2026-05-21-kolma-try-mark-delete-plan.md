# S34 Try 마킹 해제 API/UI 계획

## 목표
- API에 저장된 의미 Try 마킹을 화면에서 해제해도 서버 상태와 화면 상태가 함께 일치하게 한다.
- 연구자가 잘못 마킹한 Try를 다시 일반 Try로 돌릴 수 있게 한다.

## 범위
- `DELETE /projects/tries/:tryId/marks` API 추가
- 해당 Try의 마킹을 삭제하고 감사 로그 기록
- 웹에서 API-backed 마킹 Try 해제 시 delete API 호출
- API 성공 시 프로젝트 의미 Try 목록과 카운트에서 제거
- API 실패 시 기존 로컬 실패 안내 유지

## 제외
- 특정 mark id 단위 삭제
- 마킹 이력 보관/복구
- 삭제 확인 모달
- 다중 마킹 표시

## 구현 순서
1. API 서비스/컨트롤러 실패 테스트 추가
2. 웹 마킹 해제 실패 테스트 추가
3. 실패 테스트 확인
4. API 마킹 삭제 구현
5. 웹 마킹 해제 API 연동 구현
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

- [x] 전체 검증

## 성공 기준
- API-backed 마킹 Try를 다시 누르면 `DELETE /projects/tries/:tryId/marks`가 호출된다.
- 삭제 성공 후 해당 Try는 일반 Try로 표시된다.
- 프로젝트 의미 Try 목록과 의미 Try 카운트가 즉시 갱신된다.
- 삭제 실패 시 로컬 실패 안내가 표시되고 UI는 안전하게 유지된다.
