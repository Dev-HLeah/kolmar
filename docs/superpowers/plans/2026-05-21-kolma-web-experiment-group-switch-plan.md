# S29 프로젝트 실험 그룹 추가/전환 계획

## 목표
- 연구자가 제품 개발 프로젝트 안에서 실험 그룹을 나누고 그룹별 Try를 관리할 수 있게 한다.
- 기존 API의 실험 그룹 생성 기능을 프로젝트 상세 화면에 연결한다.

## 범위
- 프로젝트 상세 조회의 여러 실험 그룹을 화면에서 선택 가능하게 표시
- 선택한 그룹의 Try 목록, Try 추가, 테스트 결과 등록, 배합 편집을 그룹별로 분리
- 프로젝트 상세 화면에서 실험 그룹명과 목적을 입력해 새 그룹 생성
- API 실패 시 새 그룹을 로컬 화면에 임시 반영하고 안내 유지

## 제외
- 실험 그룹 수정/삭제/순서 변경
- 그룹 간 Try 이동
- 그룹별 권한 분리
- 그룹별 통계 차트

## 구현 순서
1. 웹 프로젝트 상세 그룹 전환 실패 테스트 추가
2. 웹 프로젝트 상세 그룹 생성 실패 테스트 추가
3. 실패 테스트 확인
4. 프로젝트 상세 상태를 단일 Try 배열에서 그룹 배열 기반으로 변경
5. 그룹 선택 UI와 그룹 생성 입력 UI 구현
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 프로젝트 상세 화면에서 여러 실험 그룹을 선택할 수 있다.
- 그룹을 전환하면 해당 그룹의 Try 목록, 결과 등록 대상, 배합 편집 대상이 함께 바뀐다.
- 새 그룹 생성 시 `POST /projects/:projectId/groups`를 호출하고 생성된 그룹으로 전환한다.
- API 실패 시 로컬 그룹이 생성되고 사용자에게 로컬 반영 안내가 표시된다.
