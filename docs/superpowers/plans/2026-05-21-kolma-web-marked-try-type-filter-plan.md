# S33 프로젝트 의미 Try 유형 필터 계획

## 목표
- 프로젝트 단위 의미 Try 목록에서 마킹 유형별로 후보를 빠르게 좁혀 볼 수 있게 한다.
- 최종 후보, 문제 발견, 재검토 필요 같은 의사결정 항목을 active group 전환 없이 확인하게 한다.

## 범위
- 프로젝트 의미 Try 섹션에 `프로젝트 의미 Try 유형 필터` 선택 추가
- `전체 유형` 또는 특정 마킹 유형으로 프로젝트 의미 Try 표만 필터링
- 기존 그룹별 Try 목록, 상태 필터, 의미 있는 Try만 보기 필터는 유지
- 필터 결과가 없을 때 빈 상태 문구 표시

## 제외
- API 필터 파라미터 추가
- 마킹 유형별 통계/차트
- 복수 유형 선택
- 프로젝트 간 의미 Try 통합 보기

## 구현 순서
1. 마킹 유형 필터 실패 테스트 추가
2. 실패 테스트 확인
3. 프로젝트 의미 Try 유형 필터 상태와 파생 목록 추가
4. 프로젝트 의미 Try 섹션에 필터 UI와 빈 상태 처리 구현
5. 전체 검증 및 브라우저 확인
6. 한국어 커밋 메시지로 커밋 및 푸시

- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 연구자는 프로젝트 의미 Try 목록을 전체 또는 특정 마킹 유형으로 좁혀 볼 수 있다.
- 유형 필터는 프로젝트 의미 Try 표에만 적용된다.
- 기존 그룹별 Try 목록과 Try 상태 필터는 기존처럼 동작한다.
- 필터 결과가 없으면 명확한 빈 상태 문구가 표시된다.
