# S40 최근 사용 원료 입력 UX 계획

## 목표
- 연구자가 반복 입력하는 원료명을 최근 사용 목록에서 바로 선택해 배합표에 반영할 수 있게 한다.
- 입력 편의성을 높이되 API, DB, 인증 구조는 변경하지 않는다.

## 범위
- 배합표 컴포넌트에서 최근 사용 원료 목록 표시
- 최근 원료 선택 시 첫 번째 빈 원료명 행에 자동 입력
- 빈 행이 없으면 새 행을 추가해 원료명 입력
- 브라우저 로컬 저장소에 최근 원료명 최대 8개 저장
- 중복 원료명은 한 번만 보관하고 최신 입력을 앞쪽에 배치

## 제외
- 서버 원료 사전 API
- 원료 자동완성 검색
- 단위 변환 자동화
- 사용자 계정별 서버 저장

## 구현 순서
1. FormulaInputTable 실패 테스트 추가
2. 실패 테스트 확인
3. 최근 원료 로컬 저장/선택 구현
4. 컴포넌트 스타일 추가
5. 웹 검증
6. 한국어 커밋 메시지로 커밋 및 푸시

## 성공 기준
- 로컬 저장소에 저장된 최근 원료가 `최근 사용 원료` 영역에 표시된다.
- 최근 원료 버튼을 누르면 첫 번째 빈 원료명 입력칸에 값이 채워진다.
- 입력된 원료명은 중복 없이 최근 목록에 저장된다.
- 전체 웹 테스트와 빌드가 통과한다.
