# S26 관리자 감사 로그 조회 계획

## 목표
- 관리자가 핵심 변경 이력을 빠르게 확인할 수 있게 한다.
- S25에서 저장하기 시작한 `AuditLog` 데이터를 API와 웹 화면으로 연결한다.

## 범위
- `GET /audit-logs` API 추가
- 관리자 역할 전용 API 접근 제어
- 최근 50건, 최신순 조회
- 웹 `운영 로그` 메뉴 추가
- 관리자 선택 시 운영 로그 화면에서 감사 로그 테이블 표시
- API 장애 시 로컬 안내 표시

## 제외
- 감사 로그 필터/검색
- 로그 상세 모달
- 사용자 actor 표시
- 권한 변경 로그 기록

## 구현 순서
1. API 감사 로그 서비스/컨트롤러 실패 테스트 추가
2. 웹 운영 로그 화면/내비게이션 실패 테스트 추가
3. 실패 테스트 확인
4. API audit module 구현 및 AppModule 연결
5. 웹 AuditLogPage 구현 및 라우팅/내비게이션 연결
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] API 감사 로그 테스트 추가
- [x] 웹 운영 로그 테스트 추가
- [x] 실패 테스트 확인
- [x] API 구현
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- API는 `createdAt desc`, `take 50`으로 조회한다.
- API 컨트롤러는 관리자 역할만 요구한다.
- 관리자 역할 선택 시 사이드바에 `운영 로그`가 표시된다.
- 운영 로그 화면은 action, summary, target, createdAt을 표시한다.
