# S23 안전/규제 신호 추천 연결 계획

## 목표
- 배합 후보 생성 응답에 `safetySignals`를 추가한다.
- 독성 확정 판정이 아니라 과량, 관능, 조합 검토가 필요한 신호를 연구자에게 노출한다.
- 이후 식약처, 특허, 논문, 내부 실험 데이터 신호를 같은 구조로 합칠 수 있게 한다.

## 범위
- API `POST /recommendations/draft-tries` 응답에 안전/규제 신호 배열 추가
- 입력된 원료명, 함량, 단위, 메모 기반의 결정적 1차 신호 생성
- 웹 대시보드 추천 결과 영역에 안전/규제 신호 카드 표시
- API 장애 시 로컬 후보 초안에도 최소 신호 표시

## 제외
- 실제 독성 판정
- 외부 공공 API 실시간 조회
- Supabase 저장
- 임상/법적 판정 문구

## 구현 순서
1. API 서비스 테스트에 `safetySignals` 기대값 추가
2. 웹 대시보드 테스트에 신호 렌더링 기대값 추가
3. API 서비스에 안전 신호 생성 헬퍼 추가
4. 웹 타입, 로컬 fallback, 렌더링 UI 추가
5. API/웹 테스트와 빌드 검증
6. 한국어 커밋 메시지로 커밋 및 푸시

- [x] 로컬 브라우저 화면 검증

## 신호 기준
- 아연 고함량 입력: 일일 섭취량 상한 검토 신호
- 비타민 C 또는 산미 메모: 관능/산미 안정성 신호
- 비타민 C + 아연 동시 입력: 조합 검토 신호

## 문구 원칙
- "독성입니다"처럼 확정 표현을 쓰지 않는다.
- "검토 필요", "가능성", "확인 필요" 중심으로 표시한다.
