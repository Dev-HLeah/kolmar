# S27 Try 배합 정보 편집 계획

## 목표
- 연구자가 프로젝트 상세 화면에서 Try별 배합 정보를 선택적으로 수정 저장할 수 있게 한다.
- Try는 완성 처방이 아니므로 제목, 제형, 제조 공정, 메모, 원료 행을 부분 입력해도 저장 가능하게 한다.

## 범위
- `PATCH /projects/tries/:tryId` API 추가
- Try 기본 정보 부분 업데이트
- 원료 목록이 전달되면 기존 원료를 새 행으로 교체
- Try 수정 감사 로그 기록
- 웹 프로젝트 상세 화면에 Try 배합 정보 편집 영역 추가
- API 실패 시 로컬 반영 안내 유지

## 제외
- Try 상세 별도 라우트
- 원료 행 단위 개별 수정 API
- 파일 첨부
- AI 자동 배합 생성
- 독성/규제 재계산 자동 트리거

## 구현 순서
1. API 서비스/컨트롤러 실패 테스트 추가
2. 웹 API 클라이언트 PATCH 실패 테스트 추가
3. 웹 프로젝트 상세 Try 배합 저장 실패 테스트 추가
4. 실패 테스트 확인
5. API DTO, 서비스, 컨트롤러 구현
6. 웹 클라이언트와 프로젝트 상세 편집 UI 구현
7. 전체 검증 및 브라우저 확인
8. 한국어 커밋 메시지로 커밋 및 푸시

## 성공 기준
- API는 `PATCH /projects/tries/:tryId`로 Try 선택값을 업데이트한다.
- 원료 배열이 전달되면 기존 Try 원료를 교체한다.
- 수정 성공 시 `FORMULA_TRY_UPDATED` 감사 로그가 남는다.
- 프로젝트 상세 화면에서 Try를 선택하고 배합 정보를 저장할 수 있다.
- 저장값은 비어 있는 필드를 허용한다.
