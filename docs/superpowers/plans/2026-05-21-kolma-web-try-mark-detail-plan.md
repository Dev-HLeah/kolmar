# S32 Try 마킹 유형/사유 입력 계획

## 목표
- 연구자가 의미 있는 Try를 단순 마킹이 아니라 의사결정 이력으로 남길 수 있게 한다.
- 프로젝트 단위 의미 Try 목록에서 마킹 유형과 사유를 함께 확인할 수 있게 한다.

## 범위
- 프로젝트 상세 화면에 `마킹 유형`, `마킹 사유` 입력 추가
- Try 마킹 API 호출 시 선택한 `type`, 입력한 `reason` 전달
- 프로젝트 의미 Try 표에 마킹 유형과 사유 표시
- 기존 의미 Try 필터와 마킹 카운트 유지

## 제외
- 마킹 해제 API
- 여러 마킹 이력 전체 표시
- 마킹 유형별 필터
- 별도 승인/검토 워크플로

## 구현 순서
1. 마킹 유형/사유 전달과 표시 실패 테스트 추가
2. 실패 테스트 확인
3. Try 마킹 데이터 모델을 웹에서 유형/사유까지 보존하도록 변경
4. 마킹 입력 UI와 프로젝트 의미 Try 표시 컬럼 구현
5. 전체 검증 및 브라우저 확인
6. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 연구자는 Try 마킹 전에 유형과 사유를 선택/입력할 수 있다.
- API 요청은 선택한 마킹 유형과 사유를 전달한다.
- 프로젝트 의미 Try 목록은 그룹, Try, 목적, 상태, 마킹 유형, 사유를 표시한다.
- 기존 그룹별 Try 목록, 필터, 테스트 결과 이력은 유지된다.
