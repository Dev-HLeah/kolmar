# S36 Try 기반 제품 등록 결과 링크 계획

## 목표
- 프로젝트 상세 화면에서 Try를 제품으로 등록한 직후 등록된 제품 상세로 이동할 수 있게 한다.
- 연구자가 제품 등록 성공 여부를 확인하고 후속 수정/조회 흐름으로 바로 이어갈 수 있게 한다.

## 범위
- 제품 등록 성공 응답의 `id`, `name`을 웹 상태로 보관
- 성공 안내에 `/products/:productId` 링크 표시
- 새 등록 시 마지막 등록 제품 링크를 갱신
- API 실패 또는 로컬 Try 등록 실패 시 기존 로컬 안내 유지

## 제외
- 제품 등록 후 자동 라우팅
- 등록 제품 목록을 프로젝트 상세에 누적 표시
- 제품 수정 화면 추가
- API 변경

## 구현 순서
1. 웹 제품 등록 결과 링크 실패 테스트 추가
2. 실패 테스트 확인
3. 프로젝트 상세 화면에 마지막 등록 제품 링크 상태와 렌더링 추가
4. 전체 검증 및 브라우저 확인
5. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 제품 등록 성공 후 `등록 제품 보기` 링크가 표시된다.
- 링크 href는 `/products/<createdProductId>` 형식이다.
- 성공 안내에는 등록 제품명이 표시된다.
- API 실패 시 등록 제품 링크가 표시되지 않는다.
