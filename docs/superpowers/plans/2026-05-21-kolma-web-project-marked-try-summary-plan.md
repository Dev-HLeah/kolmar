# S31 프로젝트 의미 Try 모아보기 계획

## 목표
- 연구자가 여러 실험 그룹에 흩어진 의미 있는 Try를 프로젝트 단위로 한 번에 볼 수 있게 한다.
- active group을 전환하지 않아도 후보, 테스트 완료, 유망 Try를 빠르게 찾을 수 있게 한다.

## 범위
- 프로젝트 상세 화면에 `프로젝트 의미 Try` 섹션 추가
- 모든 실험 그룹의 마킹된 Try를 그룹명, Try 번호, 제목, 상태와 함께 표시
- 기존 active group Try 목록과 마킹/상태 필터는 유지
- 마킹 추가 시 프로젝트 의미 Try 섹션도 즉시 갱신

## 제외
- 별도 API 호출 추가
- 마킹 유형/사유 편집
- 마킹 해제 API
- 프로젝트 간 통합 의미 Try 목록
- 통계/차트

## 구현 순서
1. 여러 그룹의 마킹 Try를 프로젝트 단위로 표시하는 실패 테스트 추가
2. 실패 테스트 확인
3. 프로젝트 전체 마킹 Try 파생 데이터 추가
4. 프로젝트 의미 Try 섹션과 표 구현
5. 전체 검증 및 브라우저 확인
6. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 프로젝트 상세 화면은 모든 그룹의 마킹 Try를 한 섹션에 표시한다.
- 각 행에는 그룹명, Try 번호, 제목, 상태가 표시된다.
- 마킹된 Try가 없으면 빈 상태 문구를 표시한다.
- 기존 그룹별 Try 목록, 상태 필터, 테스트 결과 이력은 유지된다.
