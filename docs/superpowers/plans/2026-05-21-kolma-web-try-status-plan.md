# S30 Try 상태 관리 계획

## 목표
- 연구자가 프로젝트 상세 화면에서 Try의 진행 상태를 바로 확인하고 변경할 수 있게 한다.
- 테스트 예정, 테스트 완료, 후보, 폐기 등 상태별로 Try를 빠르게 좁혀 볼 수 있게 한다.

## 범위
- 프로젝트 상세 API 응답의 `status` 값을 웹 Try 상태로 매핑
- Try 목록에 상태 표시
- Try 상태 필터 추가
- Try 배합 저장 시 상태 변경값을 `PATCH /projects/tries/:tryId`에 포함
- API 실패 시 상태 변경도 로컬 화면에 임시 반영하고 안내 유지

## 제외
- 상태 변경 전용 API
- 상태 변경 이력 화면
- 상태별 권한 제한
- 칸반 보드
- 마킹 유형과 상태의 자동 동기화

## 구현 순서
1. 웹 프로젝트 상세 상태 표시/필터 실패 테스트 추가
2. 웹 프로젝트 상세 상태 저장 실패 테스트 추가
3. 실패 테스트 확인
4. TryRow와 API Try 타입에 상태 필드 추가
5. 상태 라벨, 상태 필터, 상태 선택 UI 구현
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- Try 목록에서 각 Try의 상태를 볼 수 있다.
- 상태 필터를 선택하면 해당 상태의 Try만 표시된다.
- Try 배합 저장 시 상태 변경값이 API payload에 포함된다.
- API 응답 또는 로컬 fallback에서도 변경된 상태가 화면에 반영된다.
