# S25 핵심 변경 이력 감사 로그 계획

## 목표
- 제품/처방, 프로젝트, try, 테스트 결과, try 마킹 변경에 감사 로그를 남긴다.
- PoC 단계에서는 무거운 결재가 아니라 추적성 확보에 집중한다.
- 이후 관리자 운영 로그 조회 화면과 권한 변경 로그로 확장할 수 있는 DB 구조를 만든다.

## 범위
- Prisma `AuditLog` 모델 추가
- 제품 생성 로그
- 프로젝트 생성 로그
- 실험 그룹 생성 로그
- try 생성/삭제 로그
- 테스트 결과 등록 로그
- try 마킹 로그

## 제외
- 관리자 로그 조회 UI
- 권한 변경 UI
- 사용자 식별 기반 actor 추적
- 별도 큐/Redis 기반 비동기 로그 처리

## 구현 순서
1. 제품 생성 감사 로그 실패 테스트 추가
2. 프로젝트/try 변경 감사 로그 실패 테스트 추가
3. 실패 테스트 확인
4. Prisma schema에 `AuditLog` 추가
5. 제품/프로젝트 서비스에 로그 기록 추가
6. Prisma validate/generate, API lint/test/build 검증
7. 한국어 커밋 메시지로 커밋 및 푸시

## 감사 로그 필드
- `action`: 변경 행위 코드
- `targetType`: 변경 대상 종류
- `targetId`: 변경 대상 id
- `summary`: 사람이 읽을 수 있는 요약
- `metadata`: 검색/분석용 보조 데이터
- `createdAt`: 기록 시각
