# S28 Try 테스트 결과 이력 표시 계획

## 목표
- 연구자가 프로젝트 상세 화면에서 Try별 테스트 결과 이력을 바로 확인할 수 있게 한다.
- 테스트 결과 등록 후 화면에 즉시 반영해 실험 기록 누락을 줄인다.

## 범위
- 프로젝트 상세 조회의 Try 테스트 결과를 최신순으로 제공
- 웹 프로젝트 상세 화면에 `테스트 결과 이력` 표 추가
- API에서 불러온 테스트 결과 표시
- 새 테스트 결과 등록 성공 시 이력 표에 즉시 추가
- API 실패 시 로컬 화면에 입력 결과를 임시 표시하고 안내 유지

## 제외
- 테스트 결과 수정/삭제
- 결과 상세 모달
- 항목별 필터/검색
- 파일 첨부
- 통계 차트

## 구현 순서
1. API 프로젝트 상세 조회 테스트 결과 최신순 실패 테스트 추가
2. 웹 테스트 결과 이력 표시 실패 테스트 추가
3. 실패 테스트 확인
4. API include 정렬 구현
5. 웹 TryRow 테스트 결과 매핑, 등록 후 상태 반영, 이력 표 구현
6. 전체 검증 및 브라우저 확인
7. 한국어 커밋 메시지로 커밋 및 푸시

## 진행 상태
- [x] API 실패 테스트 추가
- [x] 웹 실패 테스트 추가
- [x] 실패 테스트 확인
- [x] API 구현
- [x] 웹 구현
- [x] 전체 검증
- [x] 브라우저 확인

## 성공 기준
- 프로젝트 상세 API는 Try의 테스트 결과를 `createdAt desc`로 포함한다.
- 프로젝트 상세 화면은 Try 번호, 시험 목적, 측정 항목, 측정값, 판정, 메모를 표로 표시한다.
- 테스트 결과 등록 후 새 결과가 이력 표에 즉시 표시된다.
- API 실패 시에도 로컬 입력 결과가 화면에 남고 안내 문구가 표시된다.
